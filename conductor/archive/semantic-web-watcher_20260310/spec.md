# Specification: Semantic Web Watcher Monitoring

## 1. Overview
本 Track 旨在提升 `web_watcher` 的主动监控能力，从基于字节流的哈希比对（Hash Diff）升级为基于大语言模型（LLM）的**语义监控（Semantic Monitoring）**。系统将能识别网页内容的实质性变动（如主旨、事实、核心实体），而非仅仅因为格式、时间戳或不可见字符的微小变化而触发告警。

## 2. Functional Requirements
- **网页内容语义识别**：集成 LLM 对新旧抓取到的网页内容进行对比。
- **多维度变动识别**：识别核心主旨变动、特定实体（如价格、日期、关键人物）的变动。
- **深度分析触发**：每次监控周期均发起深度全量分析。
- **自适应内容抓取**：结合 `browser_tool` 确保抓取到的内容是 JS 渲染后的准确文本。
- **多样化输出报告**：生成包含 Diff 对比、增量摘要以及预警信息的综合报告。

## 3. Non-Functional Requirements
- **SSRF 防护**：严格遵守现有的私有网络访问限制。
- **Token 消耗优化**：对超长网页进行分片摘要后再对比。
- **延迟控制**：确保监控分析在 30 秒内得出结论。

## 4. Acceptance Criteria
- [ ] 网页内容中非语义变动（如刷新时间戳、随机 ID）不会触发变动警报。
- [ ] 核心事实变更（如“原价 $99”变为“现价 $49”）能准确识别并总结。
- [ ] 任务触发后能自动生成一份准确的变动对比摘要。
- [ ] 能在 `CronJob` 模型中配置并持久化监控的元数据。

## 5. Out of Scope
- 对需要复杂登录或强验证（如滑动验证码）的页面支持。
- 对网页内非文字（如视频、复杂图片内的微小视觉变化）的监控。
